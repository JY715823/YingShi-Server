# Current Task: Stage 12.3 - 媒体管理压测联调支持

## 背景

Android 进入 Stage 12.3，重点压稳上传、媒体关系、删除、恢复、撤销等复杂链路。Server 默认不做大改，只在接口契约或返回数据不满足 Android 稳定刷新时做最小修正。

## 目标

支持以下联调：

1. 上传成功后返回完整媒体字段
2. 发成新帖子后返回新帖子和媒体关系
3. 加入已有帖子后关系查询正确
4. 目录删只移除当前帖子关系
5. 系统删移除全局媒体并进入 app 回收站
6. 空帖保护错误码清晰
7. 回收站恢复后列表查询正确
8. 24h 撤销接口或现有恢复入口语义清晰

## 不做内容

- 不做 OSS
- 不做转码
- 不改权限体系
- 不改回收站业务规则
- 不做 WebSocket / 推送
- 不做多端实时同步

## 验收

1. 上传接口返回字段完整。
2. 媒体关系增删查结果正确。
3. 空帖保护错误码清晰。
4. 删除 / 恢复后列表查询正确。
5. 如修改 Server，mvnw test 通过。
## Stage 12.3 补充

- 服务端主收口是两点：
  - `/api/media/feed` 不再返回未关联到任何有效帖子的孤立媒体，避免上传失败污染照片流。
  - 目录删 / 系统删前增加空帖保护，拒绝把有效帖子删成空帖。
- 对应联调时，客户端应把上传成功与挂帖成功分开处理，并消费服务端 409 冲突提示。
